# 宇宙群英传 策略曲线分析与优化报告

> **分析日期**: 2026-05-18  
> **分析方法**: 基于源码公式推导 + 数学模拟 + 与 MFC 原版对比  
> **范围**: 经济增长曲线、科技研发节奏、军事平衡、外交策略、纪元转换、胜利路径

## 一、经济增长曲线分析

### 1.1 采矿产出公式

```
output = floor((workersPerStope + leaderBonus) × miningWeight / 2) × treacheryFactor
       = floor((workers/stopeCount + leader.economy/20) × weight / 2) × max(1, 100-treachery) / 100
```

**初始状态模拟**（1 采矿场，65 工人，ratio=30/30/30）:

| 回合 | 工人 | 矿工 | 产出/回合 | 累计资源 |
|------|------|------|-----------|----------|
| 1 | 65 | 21 | floor(21×2/2)×1.0 = 21 | 121 |
| 10 | 75 | 25 | 25 | ~350 |
| 50 | ~200 | ~66 | min(66, 100, 星球剩余) | 受星球枯竭限制 |

**问题**: 地球初始资源 500，采矿每回合消耗 ~20，约 25 回合就会枯竭。之后需要殖民其他星球才能继续采矿。但殖民需要舰队→需要经济→需要工厂→需要资源，形成**死锁**。

### 1.2 工厂产出公式

```
ecoOutput = floor((workersPerFactory + leader.economy/30) × factoryWeight / 2) × treacheryFactor
资源消耗 = ecoOutput × 2（质能转换可免除）
```

**关键问题**: 工厂每生产 1 经济消耗 2 资源。初期资源有限时，工厂实际是在**消耗**资源换经济。当资源不足时，`add = floor(resource/2)`，经济增长被严重压缩。

### 1.3 经济曲线评估

```
                经济
  500 ┤
      │                              ╱ 有行星发动机 (×5)
  400 ┤                           ╱
      │                        ╱
  300 ┤                     ╱
      │                  ╱
  200 ┤              ╱               ╱ 基础增长
      │          ╱              ╱
  100 ┤      ╱ (建设投入期) ╱
      │  ╱             ╱
    0 ┤╱──────────╱
      └──────────────────────────────
       0   50  100  150  200  250  回合
```

**结论**: 经济曲线呈现明显的**J型增长**——前 50 回合几乎停滞（建设投入期），50-100 回合缓慢增长，100+ 回合若有行星发动机则爆发。但问题在于：

> [!WARNING]
> **经济死亡陷阱**: 前 30 回合资源枯竭→工厂停转→经济停滞→无法建造→无法殖民→无法获取新资源。玩家陷入不可逆的贫困循环。

## 二、科技研发节奏分析

### 2.1 研发速度公式

```
progress = min(100, (10 + leader.science/10) × max(1, 100-treachery)/100)
智子封锁时: progress = max(1, floor(progress/10))
```

**无部长时**: 每回合进度 = 10（无逃亡惩罚时）

| 科技名 | totalWorkload | 无部长耗时 | 有强部长(sci=100)耗时 |
|--------|---------------|------------|---------------------|
| 天文观测 | 60 | 6 回合 | 3 回合 |
| 50光年远镜 | 100 | 10 回合 | 5 回合 |
| 行星发动机Ⅰ型 | 250 | 25 回合 | 13 回合 |
| 黑域生成 | 500 | 50 回合 | 25 回合 |
| 数字方舟 | 400 | 40 回合 | 20 回合 |

### 2.2 科技路径到胜利条件的距离

**流浪胜利** (最短路径):
```
核聚变推进(120) → 重元素聚变(200) → 行星发动机Ⅰ型(250) → Ⅱ型(300) → Ⅲ型(400) 
                                                         = 总工作量 1270
流浪地球计划(200) → 新家园选址(300) = 总工作量 500
合计: 1770 工作量 ÷ 10/回合 = ~177 回合
```

**数字永生胜利**:
```
数字文明(120) → 数字方舟(400) = 520 工作量 ÷ 10/回合 = ~52 回合
```

**黑域胜利** (PHYSICS 路径):
```
维度物理(150) → 曲率驱动理论(350) → 二向箔防御学(400) → 黑域生成(500)
= 1400 工作量 ÷ 10/回合 = ~140 回合
```

> [!IMPORTANT]
> **数字永生是唯一可行的快速胜利路径**，仅需 ~52 回合。流浪胜利需要 ~177 回合，但纪元 200→201 转为威慑纪元，玩家有充足时间。黑域胜利需 ~140 回合。**征服胜利和威慑胜利**依赖外交/军事而非科技，理论上更快但风险更高。

### 2.3 智子封锁影响分析

| 阶段 | 年份 | 封锁状态 | 实际进度 | 影响 |
|------|------|----------|----------|------|
| 前期 | 0-9 | 无封锁 | 10/回合 | 正常 |
| 中前期 | 10-30 | 封锁中 | 1/回合 | **科研几乎停滞** |
| 中期 | 30+ | 若完成550W量子 | 10/回合 | 恢复正常 |

**问题**: 550W量子计算机在 INFORMATION 树中，路径为:
```
数字文明(120) → 数字生命研究(180) → 意识上传(250) → 550W量子计算机(350)
= 900 工作量
```
在封锁下（1/回合），完成需 **900 回合**——这是不可能的。

> [!CAUTION]
> **智子封锁死锁**: 要解除封锁需 550W 量子计算机，但在封锁下研发它需 900 回合。智子工程同样在封锁影响下。这意味着**玩家一旦进入封锁就几乎无法解除**，科研永久停滞。

**修复建议**: 
1. 智子封锁不应影响 INFORMATION 树（信息技术非"基础物理"）
2. 或将封锁减速从 `/10` 改为 `/3`（仅降低而非近乎停止）
3. 或提供不依赖科技的解锁路径（事件/外交）

## 三、军事平衡分析

### 3.1 异星攻击威胁时间线

| 文明 | 类型 | 首次可能攻击 | 攻击概率 | 到达时间 |
|------|------|-------------|----------|----------|
| 三体 | 猎手 | 第 1 回合 | 18% | 6 回合 |
| 歌者 | 清理者 | 第 1 回合 | 12% | 8 回合 |
| 边缘世界 | 扩张 | 第 1 回合 | 10%(殖民) | 即时 |
| 机会型* | 机会 | 第 1 回合 | 25%(低威慑时) | 4 回合 |

**地球初始防御力**:
```
defPower = (500 + army) × 2 = (500 + 10) × 2 = 1020
```

**三体攻击舰队战力**:
```
水滴型: 80 × 20 = 1600
强互作用探测器: 40 × 10 = 400
总战力: 2000
```

> **结论**: 三体首次攻击战力 2000 > 地球防御 1020，**地球在早期几乎必败**。战败后人口降至 30%，加速灭亡。

### 3.2 威慑值增长分析

```
威慑增长来源:
- 面壁者: (leadership + art) × 0.5 / 回合
- 执剑人: 仅影响 army，不直接增加 deterrenceValue
```

以罗辑（假设 leadership=90, art=70）为面壁者:
```
deterrence增长 = (90+70) × 0.5 = 80/回合
```

但异星 AI 检查的是 `deterrenceRate`（基于执剑人 leadership），**不是** `deterrenceValue`。这意味着:
- **面壁者增加的 deterrenceValue 对 AI 行为无影响**
- 只有执剑人的 leadership 才能抑制攻击

> [!WARNING]
> **威慑系统断裂**: `deterrenceValue` 被 UI 显示但对 AI 行为无实际影响。AI 只看执剑人 `leadership`（`calculateDeterrence` 方法）。玩家投资面壁者提升威慑值是**无效操作**。

## 四、外交策略分析

### 4.1 外交行动收益表

| 行动 | 冷却 | 收益 | 风险 |
|------|------|------|------|
| 谈判 negotiate | 3 回合 | friendship +1 | 无 |
| 贸易 trade | 3 回合 | -30 经济, +50 资源 | 经济不足时失败 |
| 挑衅 provoke | 3 回合 | friendship -1 | 关系恶化 |
| 结盟 alliance | 3 回合 | isBund=true | 需 friendship≥FRIEND |

### 4.2 外交策略路径

从 NORMAL(2) 到 VERYFRIEND(4) 需谈判 2 次，冷却共 6 回合:
```
回合 0: negotiate → FRIEND(3)，冷却 3 回合
回合 3: negotiate → VERYFRIEND(4)
回合 3: alliance → isBund=true
```

**征服胜利分析**: 需要 5 个文明全部灭亡或结盟:
- 最快路径: 全部结盟 = 5 × 6 = 30 回合（理想情况）
- 实际更长: 三体初始可能为 VERYANGRY(0)，需谈判 3 次 = 9 回合

**评价**: 外交系统**过于简单**——谈判 100% 成功，无失败概率，无代价。应增加:
1. 谈判成功率（基于文化/威慑值）
2. 贸易选项多样化（科技交换、军事合作）
3. 外交事件触发（背叛、联合进攻）

## 五、纪元转换与游戏节奏

### 5.1 纪元划分

| 纪元 | 年份范围 | 持续 | 主要特征 |
|------|----------|------|----------|
| 危机 CRISIS | 1-200 | 200 回合 | 建设发展，智子封锁 |
| 威慑 DETERRENCE | 201-260 | 60 回合 | 需执剑人，威慑平衡 |
| 广播 BROADCAST | 261-268 | **8 回合** | 极短，坐标暴露 |
| 掩体 BUNKER | 269-330 | 62 回合 | 防御建设 |
| 银河 GALAXY | 331+ | ∞ | 自由探索 |

> [!WARNING]
> **广播纪元仅 8 回合**：这个纪元的所有条件事件几乎不可能触发。`broadcast_era_dawn` 要求 `minYear: 120, epoch: BROADCAST`，但广播纪元是 year 261-268，minYear 120 虽满足，但 8 回合内事件触发概率极低（还需过 `Math.random() > 0.5` 检查）。

### 5.2 游戏节奏评估

```
紧张度
  高 ┤  ╱╲   ╱╲╱╲                   ╱╲
     │ ╱  ╲ ╱      ╲              ╱    ╲
  中 ┤╱    ╲         ╲          ╱        ╲
     │                ╲       ╱            ╲
  低 ┤                 ╲─────╱               ╲───
     └──────────────────────────────────────────
      0    50   100  150   200  250  300  350
      |← 危机纪元 →|←威→|←广→|← 掩体 →|← 银河 →|
      
      建设     攻击   威慑    ？   防御     自由
      期       高峰   博弈   空   建设     探索
```

**问题**:
1. **前 50 回合过于空洞**: 无事件、无威胁、仅资源积累
2. **50-200 回合过长**: 危机纪元 200 回合，中期容易无聊
3. **广播纪元过短**: 8 回合不够触发任何有意义的事件
4. **掩体纪元缺乏紧迫感**: 无降维打击倒计时

## 六、胜利路径可行性矩阵

| 胜利类型 | 最短理论回合 | 实际可行性 | 难度 | 瓶颈 |
|----------|-------------|-----------|------|------|
| 数字永生 | ~52 | ⭐⭐⭐⭐ 高 | 简单 | 仅需 2 科技，无其他条件 |
| 征服 | ~30 | ⭐⭐⭐ 中 | 中等 | 5文明全部结盟/灭亡 |
| 威慑 | ~201 | ⭐⭐⭐⭐ 高 | 简单 | 进入威慑纪元+有执剑人+人口>0 |
| 黑域 | ~140 | ⭐⭐ 低 | 困难 | 需大量科技，智子封锁阻碍 |
| 流浪 | ~177 | ⭐⭐ 低 | 困难 | 需两条科技线完成 |

> [!IMPORTANT]
> **策略不平衡**: 数字永生胜利仅需 52 回合且只需 2 个科技，远比其他路径简单。威慑胜利也很容易——只需存活到 201 回合并任命执剑人。而流浪和黑域胜利因智子封锁几乎不可达。

## 七、优化建议

### 7.1 经济系统优化

| 优化项 | 当前问题 | 建议方案 |
|--------|----------|----------|
| 初始资源 | 地球 500 资源约 25 回合枯竭 | 提升至 2000，或降低采矿消耗速率 |
| 无建筑基础经济 | 无工厂时每回合仅 +5 | 提升基础经济为 +15，确保早期不停滞 |
| 资源→经济转化比 | 2:1 过高 | 降低为 1.5:1 或 1:1 |
| 行星发动机加成 | ×5 过于极端 | 改为 ×2.5，渐进式 |

### 7.2 智子封锁优化

| 优化项 | 当前 | 建议 |
|--------|------|------|
| 封锁减速 | ÷10 | ÷3（仅减速而非近乎停止） |
| 封锁范围 | 所有科技树 | 仅 PHYSICS 树 |
| 解锁路径 | 仅 550W量子/智子工程 | 增加事件解锁（面壁计划成功） |
| 封锁开始 | year≥10 即生效 | 延后至 year≥30 |

### 7.3 威慑系统修复

| 优化项 | 当前 | 建议 |
|--------|------|------|
| AI 威慑检查 | 仅看执剑人 leadership | 综合 `deterrenceValue` 和执剑人 |
| 面壁者效果 | 仅增加无用的 deterrenceValue | 增加实际抑制力 |
| 威慑胜利条件 | 过于简单(有执剑人即可) | 增加最低 deterrenceValue 阈值 |

### 7.4 纪元节奏优化

| 纪元 | 当前 | 建议 |
|------|------|------|
| 危机 | 1-200 (200回合) | 1-150 (150回合) |
| 威慑 | 201-260 (60回合) | 151-220 (70回合) |
| 广播 | 261-268 (8回合) | 221-260 (40回合) |
| 掩体 | 269-330 (62回合) | 261-330 (70回合) |
| 银河 | 331+ | 331+ |

### 7.5 胜利路径平衡

| 胜利 | 当前问题 | 建议 |
|------|----------|------|
| 数字永生 | 过于简单(52回合) | 增加前置：数字文明→数字生命→意识上传→数字方舟 (总工作量 950) |
| 威慑 | 过于简单 | 增加条件：deterrenceValue≥200 且 有引力波广播 |
| 黑域 | 智子封锁不可达 | 修复封锁机制后可达 |
| 流浪 | 路径过长 | 缩减行星发动机链为 Ⅰ→Ⅱ→Ⅲ（去除中间科技） |

### 7.6 逃亡主义优化

| 当前 | 问题 | 建议 |
|------|------|------|
| 每回合 +random(0,2) | 前期增长稳定但无法控制 | 前 50 回合基础增长 ×0.3 |
| ≥100 立即失败 | 过于突然 | ≥80 触发危机事件链，≥100 失败 |
| 无主动降低手段 | 仅靠事件选项偶尔 -15 | 添加"社会维稳"部门行动 |
| `earlyGameFactor` | 前 100 回合 ×0.5 | 合理，但应扩展到 150 |

## 八、总结

### 策略曲线总体评价

| 维度 | 评分 | 评语 |
|------|------|------|
| 经济平衡 | 4/10 | 初期资源枯竭死锁，中后期行星发动机爆发过猛 |
| 科技节奏 | 3/10 | 智子封锁导致不可逆停滞，数字永生过于简单 |
| 军事平衡 | 5/10 | 早期防御不足，威慑系统断裂 |
| 外交深度 | 4/10 | 行动过于简单，100% 成功率无策略性 |
| 纪元节奏 | 5/10 | 危机过长、广播过短、节奏不均 |
| 胜利平衡 | 3/10 | 数字永生/威慑过简单，流浪/黑域过难 |
| **综合** | **4/10** | 核心系统框架完整但数值未精调，需全面平衡 |

### 最高优先级修复

1. **智子封锁减速 ÷10→÷3**：解决科研死锁
2. **初始资源翻倍**：解决经济死亡陷阱
3. **威慑系统连接 AI**：让面壁者有实际作用
4. **数字永生增加前置**：防止 52 回合速通
5. **广播纪元延长至 40 回合**：让事件有时间触发
