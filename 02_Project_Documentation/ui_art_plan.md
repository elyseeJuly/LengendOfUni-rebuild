# 《宇宙群英传》全新美术与 UI 设计方案（MFC内生演进版）

针对“无需实现电影级”、“结合当前开发环境”以及“增加白底主题”的需求，本方案**完全放弃了外部引擎（如Godot/Unity）的迁移建议**。所有视觉优化将严格限制在现有的 **MFC + DirectX 9.0 (3DPrelude模块已验证)** 技术栈内完成，以**最小化重构风险**，同时实现画质的大幅跃升。

---

## 一、 双主题视觉基调 (Dual Theme System)

> [!IMPORTANT]
> 游戏将提供“黑底/白底”一键切换功能，分别对应《三体》中两种截然不同的战略环境。两者在布局上完全一致，仅通过全局色彩贴图（Color Palette）和透明度参数进行区分。

### 主题 A：深空指令中心 (Dark Mode - 默认)
*   **背景设定**：太空舰队指挥室 / 掩体纪元深空预警站。
*   **配色方案**：纯黑 (`#000000`) 与极深蓝 (`#0A1128`) 渐变。
*   **强调色**：全息青 (`#00E5FF`)、高能橙 (`#FF5722`)。
*   **材质表现**：暗色磨砂玻璃（低透明度），带微弱的 CRT 扫描线噪点。

### 主题 B：PDC 联合研究终端 (Light Mode - 新增)
*   **背景设定**：行星防御理事会 (PDC) 总部 / 威慑纪元科研实验室。
*   **配色方案**：纯白 (`#FFFFFF`) 与浅银灰 (`#F0F4F8`)。
*   **强调色**：深海蓝 (`#0D47A1`)、科研青 (`#00B8D4`)。
*   **材质表现**：高透白毛玻璃（毛玻璃效果通过 GDI+ / D3D 高斯模糊 Shader 实现），强调清晰、冷峻的学术感。

## 二、 基于现有环境的 UI 重构路径 (MFC + D3D9)

我们不再追求 3D 物理级的渲染，而是通过 **“2D 高清序列帧 + GDI+ 半透明绘制 + 局部 D3D 叠加”** 来实现次世代观感。

### 1. 放弃标准 Windows 控件
*   **现状**：代码中大量使用标准的 `CDialog`, `CButton`, `CStatic`（如 `CEconomyDepartmentPanel` 等）。
*   **改造方案**：保留 `CDialog` 作为事件接收容器，但**拦截其重绘消息 (`OnPaint` / `OnEraseBkgnd`)**。使用 GDI+ 或直接将 D3D9 Device 绑定到这些对话框的 HWND 上，绘制预先渲染好的高质量 2D 边框和按钮贴图。
*   **星图重绘**：废弃 `CSolarSystemMap` 中的 `CButtonST` 阵列。将中央星图改造为一个整体的绘制区域，所有星球以 2D Sprite 形式绘制，公转轨迹使用抗锯齿曲线（GDI+ `DrawCurve`）绘制。

### 2. 半透明与毛玻璃 (Glassmorphism) 实现
*   在不更换引擎的前提下，纯正的实时毛玻璃极其耗费 CPU。
*   **折中方案**：背景使用静态的高清星空/实验室底图。当 UI 面板弹出时，程序在内存中截取该区域背景，应用一次快速高斯模糊算法（或降低对比度），然后在其上方绘制 15% 透明度的纯色图层（黑底用黑色，白底用白色），最后绘制面板边框和文字。这在 MFC 中完全可行且性能可控。

## 四、 核心特效的 2D 降级实现方案

不需要物理引擎，我们通过**预渲染动画视频或序列帧**来实现宏大叙事。

### 1. 水滴攻击 (碎屏特效)
*   **实现**：不使用实时 3D 碰撞。预先渲染一段带 Alpha 通道的视频或 60 帧 2D 序列帧（内容为水滴从远方冲出，击碎屏幕玻璃）。
*   **播放**：当事件触发时，在全屏的 D3D9 覆盖层上播放此动画。

### 2. 二向箔打击 (二维化)
*   **实现**：截取当前星图画面的屏幕截图（Render to Texture）。
*   **播放**：通过简单的 D3D9 Shader，让这张 2D 纹理的顶点从边缘向中心坍缩，最后变成一根白线消失。这在现有的 `3DPrelude` 模块技术储备中完全可以做到。

### 3. 流浪地球推进器
*   **实现**：地球模型使用简单的 2D 图标，当其处于“推进状态”时，在其图标尾部附加一个简单的 Additive 混合模式的蓝色锥形贴图（模拟等离子尾焰），通过代码改变该贴图的缩放比例（Scale Y）来模拟火焰跳动。

## 五、 实施计划

1.  **美术资源筹备 (1-2周)**: 
    *   绘制黑白双色基础控件素材（按钮、进度条、面板背景）。
    *   产出第一批核心角色高清立绘（去背透明 PNG）。
2.  **底层渲染改造 (2-3周)**: 
    *   修改 `CLengendOfUniDlg::OnPaint`，接管主界面的 GDI+ / D3D 绘制逻辑。
    *   实现主题切换系统（热切换预设的 Color Palette 和纹理路径）。
3.  **UI 模块剥离重绘 (3-4周)**: 
    *   逐个替换原有的对话框面板（经济、军事、星图等），用自绘逻辑替代 MFC 控件。
4.  **2D 特效接入 (1周)**: 
    *   整合水滴、二向箔的序列帧或 Shader 特效。

该方案将开发重心从“学习新引擎”转移到“利用现有代码基础实现纯 2D 美术升级”，风险极低，且完全能达到概念图中的视觉效果。
