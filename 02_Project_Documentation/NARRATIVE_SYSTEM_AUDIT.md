# 叙事反馈与随机事件系统评估报告

本报告对比了《刘慈欣群星传》Windows MFC 原版与 Web 重构版的叙事反馈系统，旨在找出重构版在策略游戏体验上的核心缺失。

## 1. 核心架构对比

| 特性 | Windows MFC 原版 (1.0) | Web 重构版 (0.5) | 评估结果 |
| :--- | :--- | :--- | :--- |
| **触发机制** | 同步阻塞式 (Modal Dialog) | 异步日志式 (History Log) | **降级**：Web版缺乏强制性的剧情介入感 |
| **反馈形式** | 角色头像 + 讲话者姓名 + 格式化对话 | 纯文本 History 列表 | **降级**：代入感极弱 |
| **随机系统** | 每回合进行概率判定 (`RunRandomEvent`) | 仅支持固定年份触发 (`checkEvents`) | **缺失**：缺乏变数和重玩价值 |
| **多段对话** | 支持 `TalkCount` (1-N段对话序列) | 仅支持单段 `tip` 描述 | **阉割**：无法表达复杂的剧情冲突 |
| **终局逻辑** | 实时监测并弹出大幅图文报纸/结局 | 无明显的 Turn-level 终局检查 | **缺失**：生死存亡的压力感消失 |

---

## 2. 详细缺陷分析

### 2.1 叙事中断感消失
在原版中，每回合结束点击“下一回合”后，如果触发事件，游戏会**强行暂停**并弹出对话框。这种“叙事中断”是策略游戏传达重要信息的关键手段。
*   **当前问题**：Web 版点击后，数字瞬间跳动，玩家必须手动去翻阅 `History` 才能看到发生了什么，导致“历史感”极差。

### 2.2 随机事件库真空
原版拥有独立的 `randomevent.ini`，包含大量突发状况（如科技突破、资源流失、外交波动）。
*   **当前问题**：Web 版的 `randomevents.json` 为空。这意味着除了固定的“古筝行动”、“月球危机”等几个主线事件外，游戏过程平淡如水，没有任何随机扰动。

### 2.3 视觉反馈链断裂
原版事件包含 `Talk0_Pic`（如 `shiqiang.bmp`, `sophon.bmp`），通过视觉符号强化角色存在。
*   **当前问题**：重构版虽然在 `events.json` 中保留了图片路径，但 UI 层级完全没有显示头像的组件。

### 2.4 数值影响的不可见性
原版中数值的变化往往伴随着一段文字解释（例如：“资源不足以推开月球，人口减半！”）。
*   **当前问题**：Web 版在 `Game.ts` 中直接硬编码了数值修改逻辑，但 UI 没有任何动态效果（如红色数字弹出、震动等），玩家很难察觉到数值变动背后的代价。

---

## 3. 待补全功能清单 (Restoration Roadmap)

### Phase 1: 建立反馈基础 (P0)
- [ ] **Modal Message System**：创建一个支持图片、标题和富文本的全局对话框组件。
- [ ] **Blocking Event Flow**：重写 `runARound`，使其在检测到事件时阻塞下一操作，直到玩家确认事件。
- [ ] **Multi-Talk Support**：恢复 `GameEvent` 承载多段对话的能力。

### Phase 2: 恢复随机性 (P1)
- [ ] **Random Event Engine**：实装 `RunRandomEvent` 逻辑，支持概率触发。
- [ ] **Event Content Migration**：补全 `randomevents.json` 数据，至少涵盖 20+ 种日常随机事件。

### Phase 3: 强化体验感 (P2)
- [ ] **Numeric VFX**：在资源变动时增加 UI 动效（如 `+100` 的绿色漂浮文字）。
- [ ] **Game Over Graphics**：根据失败/胜利类型，恢复原版的“报纸”或“结局大图”展示。

---

## 4. 结论

目前的重构版处于“有骨架无血肉”的状态。数值逻辑虽然跑通了，但**叙事反馈系统的缺失导致了策略深度的丧失**。玩家无法通过数值的变化感受到“黑暗森林”的压抑感或“危机纪元”的紧迫感。

**核心建议**：立即着手开发一套基于 React 的**沉浸式剧情对话框系统**，并将 `Game.ts` 中的事件处理逻辑从“静默写入日志”改为“主动弹窗告知”。
