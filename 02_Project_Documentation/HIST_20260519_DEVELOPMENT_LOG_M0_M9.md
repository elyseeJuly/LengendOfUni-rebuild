# 《宇宙群英传》Web 重制版：开发全历程实录 (M0-M9)

本项目记录了从 2026 年 5 月开始，将经典的《宇宙群英传》C++ (MFC) 源代码，全面降维重构为现代化 Web (Vite + TypeScript) 架构的技术路径与开发纪实。

---

## 🏗️ 阶段 M0：遗产封存与数据脱敏
**目标**：隔离旧代码，打通数据层。
- **MFC 隔离**：将所有旧版 `C++` 源代码迁移至 `windows_mfc/` 文件夹进行物理备份，确立了“原版保留，Web 重生”的开发原则。
- **数据降维**：编写 Python 脚本，将数万行的 GBK 编码 `.ini` 配置文件（星系数据、人物属性、武器原型）解析并转换为 Web 标准的 UTF-8 `JSON` 资产。
- **环境搭建**：初始化 `Vite + TypeScript` 脚手架，确立了严格类型检查与高效 HMR 的现代前端基座。

## 🧠 阶段 M1：核心逻辑扁平化重构
**目标**：彻底剥离指针，重塑内存模型。
- **去指针化**：将 C++ 中复杂的 `Person*` 或 `Star*` 指针引用全部改为基于 `ID/Name` 的字符串索引，从而实现了游戏状态的 100% `JSON` 序列化能力。
- **逻辑重构**：复刻了核心文明类（Civilization）、星系管理（StarManager）、人才管理（PersonManager）以及 5 大系科技树管理系统。
- **回合制内核**：实现 `runARound` 回合结算函数，完成了从 C++ 全局变量到 JS 对象单例（Singleton）的华丽转身。

## 🌌 阶段 M2：主界面与 Canvas 星图渲染
**目标**：视觉彻底现代化。
- **设计语言**：确立了“赛博深空”视觉风格，通过 CSS 变量实现了基于 **毛玻璃（Glassmorphism）** 面板的高端质感。
- **Canvas 渲染内核**：放弃了 MFC 死板的窗口控件，使用 HTML5 Canvas 绘制行星公转动画。
- **势力着色系统**：实现了星图亮色光晕与势力归属的动态视觉判定（金/蓝/橙三色分层）。

## 🏛️ 阶段 M3：部门系统与科技交互
**目标**：打通内政与科研闭环。
- **组件化重构**：将原版 11 个独立的部门窗口类（Army, Physics, etc.）合并为一个动态复用的 `DepartmentPanel` 模态框。
- **动态科技树**：实现了基于 CSS Grid 的自适应科技网格，科技节点的研发进度条与“研发中”闪烁光晕特效。
- **行星建设**：在右侧面板集成“建设矿场/工厂”交互，实现了点击即刻反馈的建设指令流。

## 🎖️ 阶段 M4：动态人物分配体系
**目标**：注入“群英”灵魂。
- **通用选人中枢**：开发了高度复用的 `PersonSelectPanel`。
- **人才赋能**：实现了将科学家（如丁仪）、领袖（如罗辑）分派至对应部门后的“效率放大器”逻辑（例如：科学值越高，研发越快）。
- **可用性同步**：自动处理人才在可用池与在职岗位间的状态流转，避免了多重指派冲突。

## 🚀 阶段 M5：星际舰队与战斗系统
**目标**：开启星际扩张玩法。
- **舰队模型**：引入 `Fleet` 实体，记录舰船构成、指挥官、起点、目标与抵达回合。
- **航线可视化**：在 Canvas 上渲染从出发点到终点的全息虚线航路，舰队光点随回合推移发生物理位移。
- **战斗引擎**：基于数值对撞与指挥官属性加成的 `CombatEngine` 上线，实现了战报输出与星系易主的完整逻辑。

## ♟️ 阶段 M6：宇宙社会学与终极博弈
**目标**：完美还原《三体》世界观。
- **面壁计划**：实现了面壁者的隐性威慑值积累逻辑。
- **执剑人威慑**：引入执剑人威慑度评估。异星 AI 会依据执剑人的性格（领导力/忠诚度）判定是否敢于发起全面进攻。
- **终极核按钮**：加入了【广播宇宙坐标】功能，一键触发“同归于尽”的终局结算。

## 💾 阶段 M7：持久化收尾
**目标**：真正可游玩的单机体验。
- **状态持久化**：开发了基于 `localStorage` 的存档/读档系统。
- **原型唤醒 (Hydration)**：通过 `Object.setPrototypeOf()` 技术，解决了 JSON 反序列化后 Class 成员函数丢失的痛点。
- **系统菜单**：新增 ⚙️ 齿轮图标与全息菜单，提供保存、读取、重启与关于声明。

## 🎭 阶段 M8：叙事现代化与工业级事件引擎
**目标**：注入“三体”叙事灵魂，构建史诗级动态反馈。
- **叙事架构重构**：设计并实现了工业级的 `GameEvent` JSON 体系。支持复杂的触发条件（纪元限制、概率、科技需求）、多节点对话队列及多分支抉择。
- **海量事件注入**：分 7 个批次注入了约 **150 个** 高保真随机事件。内容涵盖生存危机（资源匮乏、暴动）、科技伦理（AI 觉醒、维度坍缩）及黑暗森林博弈。
- **人物剧情全覆盖**：打破了通用叙事的单调，为 **章北海、程心、维德、罗辑、大史、面壁者、叶文洁** 等 15+ 位关键角色量身定制了专属叙事链与高保真头像映射。
- **动态因果反馈**：引入了 `Flag` (标记) 系统与 `EffectProcessor`。玩家的每一个抉择不仅影响当前的资源储备，还会埋下长线的伏笔（如“阶梯计划”的成败将直接影响后续纪元的情报获取）。

## 🛠️ 阶段 M9：工业级稳定性加固与自愈系统
**目标**：消除黑盒死机，保障大规模叙事流的绝对稳健。
- **全路径诊断 (Diagnostic Logs)**：在 `runARound` 核心逻辑中部署了步进式日志监控。每一回合的结算（资源、AI、剧情）都在历史面板中透明呈现，彻底告别了“年份不更新”的逻辑断层。
- **全局异常审计**：在应用最顶层部署了全局 `ErrorListener`，将所有代码层面的异常实时映射为游戏内的历史警告，实现了故障的瞬间秒级定位。
- **UI 状态自愈 (State Syncing)**：在 TopHUD 集成了“逻辑同步”热键。玩家可一键清理运行时死锁与剧情队列冲突，确保 UI 与引擎状态在任何极端载入环境下都能瞬间对齐。
- **存档安全性加固**：重构了序列化白名单，自动过滤瞬时剧情状态（currentEvent/isProcessing），确保存档文件永远是“干净、可读、可续作”的稳定状态。
