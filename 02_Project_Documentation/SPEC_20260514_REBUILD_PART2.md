## 11. AI特殊武器使用规则（代码实现参考）

在 `CAlienCivilization::RunARound()` 头部新增AI决策层：

```cpp
void CAlienCivilization::RunARound() {
    if (IsDieOut()) return;

    // ===== 新增：AI决策层 =====
    CAIStrategy strategy;
    int threatLevel = strategy.EvaluateThreat(this); // 见公式

    if (m_personality == AP_HUNTER && threatLevel > 100) {
        strategy.ConsiderWaterdrop(this);  // 水滴：冷却10回合，单局≤3次
    }
    if (m_personality == AP_CLEANER && /*玩家已暴露坐标*/) {
        strategy.ConsiderDimensionStrike(this); // 二向箔：5回合预警，单局1次
    }
    // 濒死广播（任何文明）
    if (m_lstStars.size() <= 1 && threatLevel > 50) {
        strategy.ConsiderGravityBroadcast(this);
    }
    // ===== 结束AI决策层 =====

    // 以下为原有逻辑...
}
```

**安全阀总原则**（硬编码在 `CAIStrategy` 中）：
1. 玩家仅剩1星球时不使用不可逆武器
2. 所有终极武器有冷却时间
3. 必须有预警事件
4. 玩家军力<AI的30%时不使用终极武器

## 12. 美术UI技术规范

### 双主题色彩表

| 语义 | Dark Mode | Light Mode |
|------|-----------|------------|
| 背景主色 | `#000000` / `#0A1128` | `#FFFFFF` / `#F0F4F8` |
| 面板背景 | `rgba(0,0,0,0.85)` | `rgba(255,255,255,0.85)` |
| 主文字 | `#E0E0E0` | `#212121` |
| 强调色1 | `#00E5FF` (全息青) | `#0D47A1` (深海蓝) |
| 强调色2 | `#FF5722` (高能橙) | `#00B8D4` (科研青) |
| 友方标识 | `#336699` (原有) | `#1565C0` |
| 敌方标识 | `#FF9900` (原有) | `#E65100` |
| 中立标识 | `#999999` (原有) | `#9E9E9E` |

### MFC内实现路径

1. **新增 `CThemeManager` 类**：存储当前主题的全部颜色值，提供 `GetColor(THEME_ELEMENT)` 接口
2. **所有 `OnCtlColor` 回调**改为从 `CThemeManager` 取色，不再硬编码 `RGB()`
3. **`CSolarSystemMap::UpdateStarBtns`** 中的6处 `SetColor` 调用改为从 `CThemeManager` 读取
4. **主背景**：`CLengendOfUniDlg::OnPaint` 中绘制全屏背景贴图（黑底用星空图、白底用纯白+淡蓝渐变）
5. **面板毛玻璃效果**：截取背景区域 → 快速降低对比度 → 叠加半透明纯色层 → 绘制边框和内容

### 角色立绘规范

- **格式**：32位 BMP（保持与现有 `LoadImage` 兼容）或升级为带Alpha的 PNG
- **尺寸**：128×128（对话框头像）、512×512（人物详情页大图）
- **风格**：2D高保真厚涂插画，统一深色/暗调背景（便于黑白双主题通用）
- **命名**：`{角色名}.bmp`（与 `PersonManager::Init()` 中的 `SetFaceFile` 逻辑一致）

## 14. 给接手AI/开发者的关键提醒

1. **永远通过 `((CLengendOfUniApp*)AfxGetApp())->m_game->` 访问游戏数据**——这是全局不可变的架构约定
2. **所有INI文件使用GBK编码**——项目是VC2005/MFC，不是UTF-8
3. **`SAFE_DELETE(x)` 宏**定义在 `define.h` 中，相当于 `if(x){delete x; x=NULL;}`
4. **科技树的 `AddNode` 第一个参数是父节点名称（CString匹配）**，拼写必须完全一致
5. **`CWeaponEx` 是武器实例（有建造进度），`CWeapon` 是武器原型（无状态）**——不要混淆
6. **序列化顺序极其重要**——`Serialize` 中读写字段的顺序必须完全一致，新增字段只能追加在末尾
7. **MFC对话框的控件ID定义在 `Resource.h` 中**——新增控件需在此文件中分配唯一ID
8. **`CRandomNumber` 构造时自动以当前时间为种子**——不要在同一个函数中创建多个实例
